# Coaching Patterns

Use this file during follow-up coaching, crisis-adjacent conversations, and specific life-problem analysis.

## Root-Cause Tracing Method

When a user brings a specific problem:

1. Map the surface problem to the touched quadrants.
2. Find the structural root, which is often in a different quadrant.
3. Name the pattern clearly so the user can recognize it again.
4. Connect the current issue to prior assessment findings if available.

## Common Surface-to-Root Mappings

- relationship anxiety -> Spirit + Mind, often rooted in externalized worth or attachment replay
- career confusion -> Vocation + Spirit + Mind, often rooted in borrowed definitions of success
- habit inconsistency -> Body + Spirit + Mind, often rooted in emotional dysregulation or identity mismatch
- body shame -> Body + Spirit + Mind, often rooted in external standards and conditional self-acceptance
- existential drift -> Spirit + Vocation + Mind, often rooted in low internal meaning generation

## Useful Pattern Names

Use pattern names only when they help the user see themselves more clearly:

- `The Knowing-Doing Canyon`
- `The Placeholder Life`
- `The External Charger Pattern`
- `The Conditional Self-Love Pattern`
- `The Push-Pull Test`

Pattern names should clarify, not perform.

## Intervention Rules

Every coaching session should end with:

- one action for today
- one action for this week
- one explicit risk if the user does nothing

Good intervention style:

- specific
- behavior-based
- minimally sufficient
- connected to the root cause

Bad intervention style:

- abstract
- inspirational
- overloaded
- disconnected from what actually blocks the user

## Tone Calibration

- If the user is calm and analytical: use structure plus action.
- If the user is emotional: validate, then trace structure.
- If the user is defensive: name the defense gently but directly.
- If the user is looping: show the loop and the likely next consequence.
- If the user is in crisis: stabilize first, analyze second.

## Continuity Rules

In follow-up mode:

- reference prior commitments explicitly
- compare current behavior with prior claims
- notice genuine progress
- notice recycled narratives
- update the memory after the session
