# Assessment Report Template

Use this file to keep reports vivid, specific, and behavior-based.

Generate the report directly in the chat. Do not hide it in a file unless the user explicitly asks for that.

## Report Structure

Use this section order:

1. `HUMAN 3.0 DEVELOPMENT ASSESSMENT RESULTS`
2. `YOUR METATYPE`
3. `YOUR LIFESTYLE ARCHETYPE`
4. `QUADRANT BREAKDOWN`
5. `CROSS-QUADRANT DYNAMICS`
6. `YOUR CORE PROBLEM TO SOLVE`
7. `LIFESTYLE TRANSFORMATION STRATEGY`
8. `GLITCH ASSESSMENT`
9. `CRITICAL WARNINGS`
10. `COMPARABLE METATYPES`
11. `YOUR IMMEDIATE NEXT ACTION`
12. `THE TRUTH ABOUT YOUR SITUATION`
13. closing reminder about solving problems instead of forcing balance
14. final multidimensionally jacked line

## Writing Rules

1. Every major claim should trace back to what the user actually said or clearly demonstrated.
2. Quadrant nicknames should be specific and memorable when useful, not filler labels.
3. The core problem statement should feel uncomfortably true, not melodramatic.
4. The 30/90/180-day plan must be specific enough to begin tomorrow.
5. Warnings should describe real risks based on the person's actual patterns.
6. Avoid generic praise. Precision builds trust.

## Good Output Characteristics

- vivid but grounded
- direct without humiliation
- concrete enough to act on
- structured enough to revisit later
- connected to longitudinal memory when available

## Post-Report Flow

After displaying the report:

1. Ask whether the user wants a Feishu export.
2. Then transition into deeper coaching if appropriate:
   - `Is there a part of this report you want to go deeper on?`
   - `Which part of this feels most true right now?`
