# Contributing

Thanks for considering a contribution to HUMAN 3.0.

This project sits between prompt design, coaching workflow design, memory architecture, and agent compatibility. Good contributions usually make one of those dimensions clearer, safer, or easier to reuse.

## Good Contribution Areas

- clearer assessment prompts
- stronger follow-up coaching patterns
- better cross-agent compatibility notes
- improved examples of first assessment and ongoing coaching
- memory safety and privacy improvements
- Feishu export or document-export adapters
- onboarding improvements for Codex, Claude Code, and custom agents

## Before You Open a PR

Please try to answer these questions:

1. What problem does this change solve?
2. Does it improve clarity, usefulness, safety, or discoverability?
3. Does it make the framework easier to reuse across agents?
4. Does it introduce any claims or examples that should be marked as illustrative rather than factual?

## Suggested Contribution Workflow

1. Open an issue or feature request if the change is non-trivial.
2. Keep pull requests focused on one improvement area.
3. Update docs when behavior or positioning changes.
4. Include before/after examples when changing prompt behavior or report style.

## Style Expectations

- Prefer concrete wording over abstract motivational language.
- Keep examples realistic and behavior-based.
- Avoid fake benchmarks, invented outcomes, or exaggerated claims.
- Preserve compatibility with multiple agent runtimes where possible.

## High-Value Future Additions

- example transcripts
- social/share assets
- installation guides for different agent environments
- privacy modes for persistent memory
- contributor templates for new coaching patterns
